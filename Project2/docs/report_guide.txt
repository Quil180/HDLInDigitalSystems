EEL4783 Project 2: Report Guidelines
====================================

Follow these instructions to fill in your one-page report.

1. Design Choice (2 pts):
   - Explain why you chose SystemVerilog (RTL) over HLS. 
   - Suggestion: "I chose RTL to have explicit control over the datapath and pipeline stages, allowing for precise timing and area optimization compared to the high-level abstractions of HLS."

2. Architecture (3 pts):
   - Draw a block diagram showing:
     - Input registers for A and B.
     - Multiplication stage (Stage 1).
     - Pipeline registers for products.
     - Summation stage (Stage 2).
     - Output register Y.
   - Mention that it is a 2-stage pipeline.

3. Verification (2 pts):
   - Describe the testbench: "A constrained randomized testbench was implemented using a transaction class to generate 20+ random test cases, including boundary values (-128, 127)."
   - Mention Assertions: "SystemVerilog Assertions (SVA) were used to automatically verify that the output matched a reference model exactly 2 clock cycles after inputs were driven."

4. Reflection (3 pts):
   - Hardest part: Likely managing the 2-cycle latency in the testbench or getting the SVA property correct.
   - Improvement: "Adding a 'valid' signal to the pipeline to indicate when the output data is ready."

Required Screenshots:
--------------------
A. Waveform Screenshot:
   - Open your simulator (e.g., Questasim).
   - Add 'clk', 'a', 'b', and 'y'.
   - Expand 'a' and 'b' to show individual elements.
   - Show a transition where 'a' and 'b' change, and 'y' updates 2 cycles later.
   - Take 3 screenshots (or 1 large one showing 3 cases).

B. Synthesis/Resource Report (For Extra Credit):
   - You must run a synthesis tool (like Xilinx Vivado or Synopsys Design Compiler) on the server.
   - Run synthesis for a non-pipelined version (you can quickly comment out the registers in dot_product.sv) and then for the pipelined version.
   - Capture the 'Timing Summary' showing the 'Slack' or 'Fmax'.
   - Capture the 'Resource Utilization' showing LUTs/Registers used.
   - Compare them: The pipelined version should have a higher Fmax (better frequency) at the cost of more registers.
